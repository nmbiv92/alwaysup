## AlwaysUp 是什么？

简单来说：它是一个把普通程序变成 Windows 系统服务的工具。

Windows 系统服务有个特性——哪怕没有用户登录，它也会在后台默默运行。AlwaysUp 就是把你的可执行文件、批处理脚本、Java 程序、快捷方式……全都"包装"成系统服务来跑。

## 它能解决什么问题？

有几个场景特别典型：

**场景一：服务器需要挂着一个非服务程序**
你有个 Node.js 脚本、一个 Python 爬虫、或者某个没有原生服务化的第三方工具，需要它开机就跑、全天不停。AlwaysUp 三步搞定，不需要写任何代码。

**场景二：程序崩了自动重启**
设置好之后，AlwaysUp 会持续监控你托管的程序。一旦崩溃、挂起、或者 CPU/内存占用超标，它会自动关掉错误弹窗然后重启，整个过程不需要人工介入。

**场景三：无人值守运行**
开机自启，无需用户登录触发。服务器重启后，程序自动恢复运行，完全不需要有人坐在那边看着。

**场景四：定时重启防内存泄漏**
有些老程序跑久了会吃内存。AlwaysUp 可以设置在凌晨业务低峰期自动重启程序，让它"保持新鲜"，避免内存问题积累成故障。
